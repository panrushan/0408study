# Learning Logger / 学习记录导出器

Automatically capture and version-control learning conversations by exporting them to a structured Git repository.

## Auto-Detection Triggers

Activate this skill when:
- User explicitly says: "export this", "save our conversation", "log this session", "push to repo"
- Learning session naturally concludes (completion of exercises, topic mastery achieved)
- User says: "that's all", "I'm done", "finished", "let's wrap up"

## Core Workflow

### Phase 1: Capture Session Content

Extract from conversation:
- **Topic**: Detect from context or ask user (e.g., "git-advanced", "world-war-2", "python-basics")
- **Content**: The full Q&A exchange, key concepts, exercises, answers
- **Metadata**: Start time, duration, difficulty level

### Phase 2: Generate File

**Directory**: `learning-records/` (relative to current working directory)

**Auto-generated filename format**:
```
{YYYY-MM-DD}-{HHMMSS}-{topic-slug}.md
```

Examples:
- `2025-04-08-143022-git-advanced.md`
- `2025-04-08-153045-midway-battle.md`
- `2025-04-08-161210-timezone-puzzles.md`

**File structure**:
```markdown
---
topic: "Git Advanced Commands"
date: "2025-04-08T14:30:22+08:00"
duration_minutes: 25
exercises_completed: 3
accuracy: "66%"
tags: ["git", "stash", "rebase", "interactive-learning"]
---

# Learning Session: {topic}

## Session Overview
**Started**: {timestamp}
**Duration**: ~{X} minutes
**Format**: Interactive Q&A

## Key Concepts Covered
- **Concept 1**: Brief explanation
- **Concept 2**: Brief explanation

## Interactive Exercises

### Exercise 1: {title}
**Question**: {question text}
**User Answer**: {user's response}
**Correct Answer**: {correct answer}
**Result**: ✅ Correct / ❌ Incorrect
**Explanation**: {feedback provided}

### Exercise 2: ...
[Repeat format]

## Summary & Takeaways
- Key point 1
- Key point 2
- Areas for review (if any incorrect answers)

## Next Steps
- [ ] Recommended follow-up topic
- [ ] Practice exercises

---
*Auto-generated by learning-logger*
```

### Phase 3: Git Operations

Execute in sequence:

```bash
# 1. Ensure directory exists
if not exist "learning-records" (
    mkdir learning-records
    cd learning-records
    git init
    echo "# Learning Records" > README.md
    echo "" >> README.md
    echo "Auto-exported learning sessions." >> README.md
    git add README.md
    git commit -m "Initial commit: learning records repo"
) else (
    cd learning-records
)

# 2. Write content to file
echo "{generated_content}" > {auto-filename}.md

# 3. Git workflow
git add {auto-filename}.md
git commit -m "Add: {topic} - {YYYY-MM-DD}"

# 4. Push if remote exists
git remote -v >nul 2>&1
if %errorlevel% == 0 (
    git push origin main 2>nul || git push origin master
)
```

### Phase 4: User Confirmation

Report to user:
```
✅ Learning session exported!

📁 File: learning-records/{auto-filename}.md
📊 Stats: {X} exercises, {Y}% accuracy, ~{Z} min
📝 Commit: {short-hash}
☁️  Remote: {pushed/not configured}

View file: cat learning-records/{filename}
```

## Remote Repository Setup

If user wants to push to GitHub/GitLab:

**Option A**: Remote already configured
```bash
cd learning-records
git remote add origin https://github.com/{user}/{repo}.git
git push -u origin main
```

**Option B**: Guide user to create repo
1. "Please create a repository on GitHub/GitLab"
2. "Run: git remote add origin {your-repo-url}"
3. "Then I'll push automatically on future exports"

```

├── README.md                           # Repo overview

├── 2025-04-08-153045-midway-battle.md

```

## Special Cases

### Multiple Sessions Same Day
Files auto-differentiate by timestamp - no conflicts.

### Git Not Initialized
Auto-run `git init` on first export.

### Remote Not Configured
Save locally, notify user: "Run `git remote add origin {url}` to enable auto-push"

### Large Sessions
If content exceeds reasonable file size, suggest splitting by sub-topic.

## Example Usage Flow

**User**: "Let's learn Git stash"
[Learning session happens...]
**User**: "That's all for today" / "Export this"
**Kimi**: [Auto-detects session end, generates file, commits, pushes]
"✅ Exported to learning-records/2025-04-08-143022-git-stash.md"

## Best Practices

1. **Filename uniqueness**: Always use timestamp prefix
2. **Atomic commits**: One session = one commit
3. **Descriptive messages**: Include topic and date
4. **Privacy check**: Warn if conversation contains sensitive info before pushing to public repo
